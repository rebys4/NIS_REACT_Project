# Презентация: Product Scope Lab

## Слайд 1. Тема

Веб-приложение для просмотра и анализа 3D моделей изделий.

## Слайд 2. Список технологий и почему они выбраны

- React: компонентный UI и гибкая экосистема.
- TypeScript: строгая типизация доменных данных и состояния.
- Three.js: рендер 3D-сцены и управление камерой/светом.
- Zustand: простое централизованное состояние без избыточного boilerplate.
- MUI: готовые UI-компоненты и тема.
- Vite: быстрый dev-сервер и сборка.

## Слайд 3. React

React применялся для построения интерфейса по FSD-архитектуре. Благодаря разделению на layers/slices легко масштабировать проект и поддерживать переиспользуемость.
Сравнение с аналогами: в сравнении с Vue и Svelte, React имеет более широкую экосистему и стандартен для многих команд.

## Слайд 4. TypeScript

TypeScript обеспечивает типобезопасность метрик, состояний сцены и интерфейсных контрактов.
Сравнение с аналогами: JavaScript проще на старте, но TypeScript надежнее при росте проекта.

## Слайд 5. Three.js

Three.js отвечает за 3D-рендер, OrbitControls, настройки материалов и света.
Сравнение с аналогами: Babylon.js также силен в 3D, но Three.js чаще используется в кастомных frontend-решениях и имеет больше примеров.

## Слайд 6. Zustand

Zustand хранит настройки viewport и результаты анализа в едином store.
Сравнение с аналогами: Redux Toolkit дает больше паттернов для крупных enterprise-сценариев, но требует больше шаблонного кода.

## Слайд 7. MUI

MUI ускоряет реализацию адаптивного интерфейса и обеспечивает визуальную консистентность.
Сравнение с аналогами: Tailwind гибок, но требует вручную собирать больше паттернов компонентов.

## Слайд 8. Vite

Vite дает быстрый HMR и простую конфигурацию сборки.
Сравнение с аналогами: Webpack мощный, но сложнее в начальной конфигурации.

## Слайд 9. Суть приложения

- Выбор модели из каталога.
- Настройка рендера (детализация, материал, свет, помощники сцены).
- Расчет метрик геометрии.
- Экспорт результатов анализа в JSON.

## Слайд 10. Скриншоты работы

- Стартовая страница: `app/docs/start-page-screenshot.svg`.
- Дополнительно рекомендуется приложить 2-3 скриншота режимов материала (Matte/Polished/X-Ray).

## Слайд 11. Заключение

Проект показал, что связка React + Three.js + Zustand хорошо подходит для интерактивной инженерной визуализации в браузере. Наиболее сильный эффект дала типизация TypeScript и структурирование FSD, благодаря чему рендер-логика и UI-логика остаются изолированными и легко расширяемыми.
