# REPORT

## Суть приложения

`Product Scope Lab` - веб-приложение для просмотра и анализа 3D-моделей изделий. Пользователь выбирает один из пресетов модели, управляет визуализацией (свет, материал, сетка, оси, bounds, детализация), а система рассчитывает геометрические метрики (число вершин, число треугольников, площадь поверхности, объем, габариты и индекс сложности). Результаты можно выгрузить в JSON для дальнейшего использования в документации или инженерных заметках.

## React

React использовался как основной UI-фреймворк: за счет компонентной модели удобно разнести страницу на `pages/widgets/features/entities/shared` и переиспользовать блоки интерфейса. В сравнении с Vue и Svelte, React дает максимально широкую экосистему и привычный для командного frontend-проекта подход с богатым набором библиотек и tooling.

## TypeScript

TypeScript применялся для типобезопасности всего проекта: описаны типы модели, метрик, состояний сцены и API стора. Это уменьшает вероятность ошибок при развитии фич и повышает предсказуемость рефакторинга. По сравнению с JavaScript без типов, TS требует больше дисциплины, но заметно выигрывает в сопровождении средних и больших проектов.

## Three.js

Three.js используется для рендера 3D-сцены: инициализация камеры, освещения, OrbitControls, отрисовка геометрии и визуальные режимы материала. Это дает гибкость низкоуровневого контроля и позволяет делать нестандартные сценарии анализа. В сравнении с Babylon.js Three.js обычно проще интегрируется в lightweight frontend-проекты и имеет очень большое сообщество и базу примеров.

## Zustand

Zustand выбран для управления состоянием рендера и анализа: стор компактный, без шаблонного кода, легко связывается с React-компонентами и хорошо подходит для локальной прикладной логики. По сравнению с Redux Toolkit он проще для старта и требует меньше инфраструктуры, что полезно для учебного и прототипного проекта.

## MUI

MUI использован для UI: карточки, стек-композиции, переключатели, формы и тема позволили быстро собрать аккуратный адаптивный интерфейс. По сравнению с чистым CSS разработка ускоряется за счет готовых компонентов и консистентности design language, при этом сохраняется возможность глубокой кастомизации.

## Vite

Vite служит сборщиком и dev-сервером. Он дает быстрый запуск, HMR и простую конфигурацию для React + TypeScript. В сравнении с классической связкой Webpack конфигурация короче и проще, а цикл разработки заметно быстрее.

## Заключение

В работе получилось собрать цельный мини-продукт: современный UI, интерактивная 3D-сцена и автоматический расчет метрик. Самый полезный опыт - сочетание Three.js с FSD-подходом и строгой типизацией: логика рендера осталась изолированной, а интерфейс и состояние масштабируются без хаоса. Из всех технологий наиболее комфортной оказалась связка `React + TypeScript + Vite`, а самым интересным в реализации - аналитика геометрии и режимы визуализации модели.
